Requisitos

Capacidades do Projeto

Esta página de requisitos do projeto serve como um guia abrangente para estabelecer e documentar as necessidades, funcionalidades e critérios essenciais que irão orientar o desenvolvimento e a entrega bem-sucedida do projeto. Aqui, você encontrará uma lista detalhada de requisitos funcionais e não funcionais, que abrangem desde as metas globais do projeto até os detalhes técnicos específicos. Estes requisitos foram cuidadosamente elaborados para garantir que todas as expectativas e objetivos do projeto sejam atendidos de forma clara e mensurável. Além disso, esta página serve como um ponto de referência para todos os membros da equipe, partes interessadas e partes envolvidas no projeto, garantindo uma compreensão comum das metas e critérios de sucesso que orientam o seu desenvolvimento e implementação

Requisitos Funcionais

Requisitos funcionais são especificações que descrevem as funções e capacidades que o projeto deve executar para atender às necessidades do usuário ou do cliente.

Mecanismos e Estrutura

  • Montagem dobsoniana;
  • O sistema dobsoniano permitirá a rotação horizontal 360º;
  • O sistema dobsoniano permitirá a inclinação vertical 90º;
  • O sistema dobsoniano permitirá girar livremente em ambos os eixos;
  • O sistema dobsoniano permitirá a sustentação com contra peso ponderado.

Hardware

  • Controlar motores de rotação, aceleração e luminosidade;
  • Monitoramento de condições de luminosidade;
  • Interfacear os aplicativos Stellarium com o Star4u.

Software

  • Comunicação entre sistemas;
  • Registro de observações;
  • Modo de operação manual;
  • Calibração automática da posição desejada;
  • Listar operações disponíveis;
  • Escolher ponto no espaço para alinhar o telescópio;
  • Escolher um objeto para ser acompanhado;
  • Escolher um script para guiar o telescópio;
  • Mostrar logs de determinado período;
  • Alertas de condições adversas.

Requisitos Não Funcionais

Requisitos não funcionais são critérios que definem como um sistema deve operar, em vez do que ele deve fazer. Eles abordam aspectos como desempenho, segurança, usabilidade e escalabilidade.

Mecanismos

  • Precisão de rotação e inclinação;
  • Tempo de resposta;
  • Estabilidade estrutural.

Hardware

  • Consumo de energia;
  • Suportar no mínimo 5 Kgf.cm;
  • Capacidade de expansão;
  • Tempo de calibração;
  • Escalabilidade;
  • Durabilidade em operações repetitivas.

Software

  • Compatibilidade com protocolos e sistemas;
  • Integridade de dados;
  • Precisão de localização;
  • Leitura e escrita de dados;
  • Facilidade de uso.
